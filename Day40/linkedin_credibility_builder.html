1. The system prompt design

Claude is instructed to act as a LinkedIn strategist who specializes in early-career profiles — that specificity matters more than a generic "resume expert" persona, because a new grad's problem (too little material) is the opposite of a senior candidate's problem (too much to trim).

  • Scope: the prompt limits Claude to LinkedIn Headline, About, Skills, and Experience/Projects — nothing about resumes or cover letters, so answers stay focused.
  • Strict output contract: Claude must return only JSON matching a fixed schema (score, assessment, suggested copy, reasoning per section). This is what lets the interface render score seals and cards instead of a wall of chat text.
  • Edge cases handled explicitly: thin/missing input → Claude still scores honestly and explains what's missing rather than inventing fake experience; off-topic input → Claude declines and restates its scope; abusive input → Claude refuses without engaging.
  • Grounding rule: Claude is told never to fabricate specific numbers, employers, or credentials the student didn't provide — it can suggest how to phrase an impact, but not invent one.

2. Why the UI choices fit this use case

  • A "credential" visual language (ink, brass, seals) instead of a generic chat box — it borrows from diplomas and transcripts, which mirrors what a new grad is actually trying to build: credibility on paper.
  • Guided form, not free chat — new grads often don't know what's worth including. Structured fields (skills, projects, achievements) prompt them for the raw material Claude needs.
  • Score seals per section make the report scannable at a glance, and collapsible cards keep the reasoning available without overwhelming the page.
  • One-click copy on every suggested block, because the entire point of the output is to be pasted directly into a LinkedIn profile.
