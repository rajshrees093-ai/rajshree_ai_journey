# Day 37 — Task Compass: Café / Restaurant Edition

## Overview
**Task Compass** is a single-file HTML/CSS/JavaScript management simulation that teaches organizational thinking — ownership, delegation, collaboration, and workflow — using a **Café / Restaurant** as the workplace context.

- **File:** `task_compass.html`
- **Tech:** Vanilla HTML, CSS, JavaScript only (no frameworks, no external libraries, fully offline)
- **Roles modeled:** Barista, Server, Kitchen Staff, Cashier, Shift Manager, Owner/GM, Busser, Supplier Relations

## Gameplay Summary

### Stage 1 — Who Owns This?
Drag/tap a role card onto the ownership slot for 3 randomly drawn tasks (from a pool of 10), such as a burnt coffee complaint or a broken espresso machine mid-rush. Each submission reveals the primary owner, the reasoning, and which roles typically assist.

### Stage 2 — Task Routing
Build the correct handoff sequence (e.g. Server → Kitchen → Cashier → Customer) for 3 tasks drawn from a pool of 5, ranging from 2-step swaps to 4-step catering requests. Submitting animates the ticket traveling through the built sequence.

### Stage 3 — Collaboration Challenge
Assign one Primary Owner plus up to 3 Supporting roles for larger situations (a viral social post, a slow-service review streak, a new competitor opening across the street), then see the ideal team, the reasoning, and a possible communication flow.

### Results Dashboard
An animated bar chart scores four dimensions — **Ownership, Delegation, Collaboration, Workflow Thinking** — followed by a written reflection: what was understood well, where responsibility tended to be over-assigned to one person, where collaboration was underestimated, and one general insight about how organizations work.

## Key Learnings
- Ownership and delegation aren't opposites — most real tasks have one accountable owner *and* a supporting cast, and the game's scoring reflects both at once rather than treating "who owns this" as a single right answer.
- Task routing reveals how much invisible structure exists in a small business — even a "simple" café has predictable escalation paths (front-line staff → shift manager → owner) depending on stakes and cost.
- Complex, ambiguous problems (a viral post, a new competitor, dropping satisfaction scores) resist single-owner thinking. The collaboration stage was the best test of whether a player defaults to "one person fixes this" versus recognizing a cross-functional response.
- Designing the reflection to avoid absolute language ("tended to," "often," "in this session") made feedback feel diagnostic rather than judgmental — useful for a learning tool aimed at building intuition, not scoring correctness.

## How to Run
1. Download `task_compass.html`.
2. Open it directly in any modern browser (Chrome, Edge, Firefox, Safari) — no server or internet connection required.
3. Play through all three stages and view the final dashboard.
